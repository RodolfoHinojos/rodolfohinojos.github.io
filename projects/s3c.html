S3C – Sistema de Control de Crédito a Cliente

Sistema desarrollado en Java como capa de validación previa a la habilitación de crédito en el ERP empresarial. Integrado directamente con Microsip mediante conexión a base de datos.

Contexto de Negocio

La organización operaba con financiamiento a plazos en dos sucursales (lote de autos y mueblería). Se requería garantizar que únicamente clientes con información completa y validada pudieran ser habilitados para crédito.

El ERP permitía operaciones comerciales, pero no gestionaba control avanzado de integridad ni validación previa de expediente.

Arquitectura General

Tecnologías

  • Lenguaje: Java
  • Base de datos: Firebird
  • Reporting: JasperReports
  • Arquitectura: Cliente-Servidor

Modelo de Integración

  • Base de datos independiente para información extendida del cliente.
  • Conexión directa a base de datos del ERP para lectura y actualización.
  • Uso de transacciones explícitas para garantizar atomicidad y consistencia.

Decisiones de Arquitectura

  • Separación de responsabilidades entre evaluación financiera y operación comercial.
  • Desacoplamiento de datos sensibles en base independiente.
  • Actualización manual controlada para evitar habilitación accidental.
  • Control de integridad previo a sincronización con ERP.

Control Transaccional

Las actualizaciones hacia el ERP se ejecutan dentro de transacciones explícitas en Firebird, garantizando:

  • Atomicidad de cambios.
  • Rollback ante inconsistencias.
  • Integridad referencial.
  • Prevención de estados intermedios inválidos.

Manejo de Concurrencia

  • Diseño multiusuario concurrente.
  • Roles: Administrador y Capturista.
  • Operación simultánea en dos sucursales.
  • Prevención de conflictos mediante control de validación previa.

Lógica de Negocio Implementada

  • Validación de campos obligatorios.
  • Detección de clientes duplicados.
  • Control de completitud de expediente.
  • Habilitación manual del estatus de crédito.

Reporting

Implementación de reportes dinámicos utilizando JasperReports para generación de documentación relacionada al expediente del cliente.

Retos Técnicos Resueltos

  • Integración segura con base de datos en producción.
  • Mantenimiento de consistencia entre sistemas desacoplados.
  • Control de integridad sin modificar lógica interna del ERP.
  • Prevención de corrupción de datos financieros.

Impacto

  • Reducción de clientes duplicados.
  • Mayor control en habilitación de crédito.
  • Estandarización de procesos entre sucursales.
  • Disminución de errores operativos.

Este proyecto demuestra experiencia en diseño de arquitectura empresarial, integración ERP, control transaccional y desarrollo backend en entornos productivos.
